## Development Workflow (ALWAYS follow this order)

### After every code change:
1. `pnpm build` — must pass with zero errors
2. **Browser test** using `agent-browser` skill BEFORE deploying:
   - Test https://learn-dsa.frontendx.dev/ (or localhost if dev server running)
   - Verify: home, /learn, /explore, /learn/sliding-window all return 200 and render content
   - Take screenshot of each page
   - Only proceed if all pages render correctly (no blank page, no 500)
3. `wrangler deploy` — only after browser test passes
4. `git add -A && git commit` — commit after every completed step with descriptive message

### Commit message format:
```
feat: <what was added>
fix: <what was fixed>
style: <UI/CSS changes>
```

### Deploy target:
- URL: https://learn-dsa.frontendx.dev
- Command: `wrangler deploy`
- Cloudflare worker name: tanstack-start-app

### Known issues to avoid:
- NEVER put `'use client'` or `"use client"` in route files — breaks SSR with 500
- NEVER use backticks inside template literal strings in article data files (use escaped string or single quotes instead)
- `@tanstack/react-virtual` breaks Workers SSR — use custom virtualizer instead
